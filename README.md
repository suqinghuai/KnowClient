# 🤖 识客宝 - 微信聊天记录智能分析程序

## 📖 项目简介

**识客宝（KnowClient）** 是一款基于大语言模型的微信聊天记录智能分析工具，旨在帮助业务人员快速、批量地分析客户聊天记录，精准评估客户的业务意向程度，并生成专业的跟踪话术。

通过 AI 技术，识客宝能够从海量聊天记录中提取关键信息，为业务决策提供数据支持，让每一位客户都能得到精准的跟进服务。

## ✨ 核心功能

### 🎯 智能意向分析
- **客户意向识别**：自动判断客户是否具有业务咨询意向
- **业务类型识别**：准确识别客户咨询的具体业务内容
- **订单状态判断**：智能区分售前和售后场景

### 💬 聊天内容深度挖掘
- **客户关注点提取**：精准捕捉客户关心的问题和疑虑
- **态度分析**：分析客户的当前态度和情绪倾向
- **聊天总结**：自动生成简洁明了的对话摘要

### 📝 专业话术生成
- **三段式跟踪策略**：提供分三段发送的跟踪话术，避免信息过载
- **个性化称呼**：自动识别客户姓氏，生成"某工"等亲切称呼
- **多轮跟进方案**：提供三次不同的跟踪话术，应对不同跟进场景

### 🔧 技术特性
- **批量处理**：支持一次性分析多个聊天记录文件
- **智能截断**：自动处理超长聊天记录，确保符合 API 限制
- **错误重试**：内置重试机制，提高分析成功率
- **日志记录**：完整的运行日志，便于调试和问题追踪
- **灵活配置**：支持自定义 API 密钥、模型名称等参数

## 🚀 快速开始

### 方式一：使用可执行文件（推荐）

1. **准备程序文件**
   - 将打包好的 `.exe` 文件放在任意文件夹中
   - 确保同一目录下有 `config2.ini` 配置文件
   - 创建 `聊天记录` 文件夹，放入待分析的 `.txt` 格式聊天文件

2. **配置 API**
   - 打开 `config2.ini` 文件
   - 填写您的 ModelScope API 密钥
   ```ini
   [DEFAULT]
   base_url = https://api-inference.modelscope.cn/v1/
   api_key = your_api_key_here  # 替换为您的API密钥
   model_name = Qwen/Qwen3-VL-8B-Instruct
   ```

3. **运行程序**
   - 双击 `.exe` 文件即可运行
   - 程序会自动分析 `聊天记录` 文件夹中的所有 `.txt` 文件
   - 分析结果会显示在控制台，并保存为 CSV 文件

### 方式二：使用 Python 源码

1. **安装依赖**
   确保已安装 Python 3.7+ 环境，运行以下命令安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

2. **配置 API**
   修改 `config2.ini` 文件，填写正确的 API 密钥和模型名称：
   ```ini
   [DEFAULT]
   base_url = https://api-inference.modelscope.cn/v1/
   api_key = your_api_key_here
   model_name = Qwen/Qwen3-VL-8B-Instruct
   ```

3. **准备聊天记录**
   - 将微信聊天记录导出为 `.txt` 格式
   - 文件命名格式：`私聊_客户昵称.txt` 或 `客户昵称.txt`
   - 将文件放入 `聊天记录` 文件夹

4. **运行程序**
   执行以下命令运行主程序：
   ```bash
   python main.py
   ```

### 打包为可执行文件

如果需要将程序打包为 `.exe` 文件，使用 PyInstaller：
```bash
pyinstaller --onefile main.py
```

打包完成后，可执行文件位于 `dist/` 目录下。

## ⚙️ 配置说明

### API 配置
- **base_url**: 兼容OpenAI格式的大模型 API 的基础地址
- **api_key**: 您的 API 密钥（从 [ModelScope]
- **model_name**: 使用的模型名称（默认：Qwen/Qwen3-VL-8B-Instruct）

### 获取 API 密钥
1. 访问大模型提供方官网
2. 注册/登录账号
3. 进入控制台，创建 API 密钥
4. 确保账户有足够的调用额度

## 📝 日志查看

程序运行后，日志文件会存储在 `logs/` 目录下，文件名格式为 `analysis_YYYYMMDD_HHMMSS.log`。

日志包含以下信息：
- 程序启动时间
- 每个文件的处理进度
- API 调用状态
- 错误和警告信息

## ⚠️ 注意事项

- 确保 `config2.ini` 文件与程序在同一目录下
- 确保 `聊天记录` 文件夹与程序在同一目录下
- API 密钥需具备足够的调用额度
- 如果聊天记录过长，程序会自动截断内容以符合 API 的限制
- 聊天记录文件必须是 `.txt` 格式
- 建议单个聊天记录文件不超过 10MB

## ❓ 常见问题

**Q: 程序运行很慢怎么办？**
A: 分析速度取决于 API 响应时间和网络状况，建议在网络良好的环境下使用。

**Q: 如何提高分析准确性？**
A: 确保聊天记录完整、清晰，包含足够的对话上下文。

**Q: 支持哪些模型？**
A: 支持所有兼容 OpenAI API 格式的大语言模型，可在配置文件中修改。

**Q: 可以分析群聊记录吗？**
A: 目前主要针对私聊记录优化，群聊记录可能需要手动调整。

**Q: 程序崩溃了怎么办？**
A: 查看 `logs/` 目录下的日志文件，了解具体错误信息，并根据提示进行调整。
